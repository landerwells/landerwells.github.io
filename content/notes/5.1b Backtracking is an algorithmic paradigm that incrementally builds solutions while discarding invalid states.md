Backtracking is an algorithmic paradigm that incrementally builds solutions
while discarding invalid states. It helps to visualize every backtracking
problem as a decision tree. Each solution is built by following one path through
the decision tree.

Backtracking recursively has a poor time complexity; it is only necessary when
enumerating an entire solution set. Not necessary when looking for a single
optimal answer.

1.  Base case
2.  Choices
3.  Constraints
4.  Backtrack

<!--listend-->

```python
def backtrack(params):
    if base_case_condition:
        save_result
        return

    for choice in choices:
        if violates_constraints:
            continue

        make_choice
        backtrack(updated_params)
        undo_choice  # Backtracking Step
```
